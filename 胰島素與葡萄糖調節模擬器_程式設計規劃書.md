# 胰島素與葡萄糖調節模擬器：程式設計規劃書

## 1. 專案摘要

製作一個給高中生使用的互動式網頁模擬器，讓學生透過兩個主要按鈕觀察血糖與胰島素之間的調節關係：

- 按鈕 A「攝取葡萄糖」：提高血液中的血糖值。
- 按鈕 B「分泌胰島素」：提高胰島素濃度，並使血糖隨時間下降。
- 血糖 Meter：持續顯示目前血糖高低及所屬區間。

本階段只做最小可運作版本（MVP）。成品是一個可直接以瀏覽器開啟的單一 `index.html`，CSS 與 JavaScript 全部內嵌，不使用框架、套件、後端或外部資源。

## 2. 教學目標

學生操作後應能理解：

1. 攝取葡萄糖後，血糖會上升。
2. 胰島素可促進血糖下降，使血糖往正常範圍移動。
3. 胰島素的作用需要時間，不是按下後立即將血糖歸零。
4. 過高與過低的血糖都不是理想狀態。

頁面需註明：此模擬器為教學用簡化模型，不可用於醫療判斷。

## 3. MVP 範圍

### 3.1 必須完成

- 單一 HTML 檔案，內含 CSS 與 JavaScript。
- 繁體中文介面。
- 支援手機、平板與電腦的響應式版面（RWD）。
- 顯示目前血糖數值，單位為 `mg/dL`。
- 以 Meter（建議使用水平色帶與指標）呈現血糖高低。
- 提供「攝取葡萄糖」按鈕。
- 提供「分泌胰島素」按鈕。
- 顯示目前胰島素的相對量，使用教學用單位，不宣稱是真實醫療濃度。
- 模擬數值會隨時間更新。
- 提供「重新開始」按鈕，恢復初始狀態。
- 顯示簡短且即時的狀態文字，例如「血糖偏高」或「血糖在正常範圍」。

### 3.2 本階段不做

- 登入、儲存進度、資料庫或後端。
- 真實病患資料與醫療劑量換算。
- 糖尿病類型、升糖素、飲食種類、運動等進階變因。
- 多頁面、歷史折線圖、音效、排行榜或測驗系統。
- 外部 JavaScript/CSS 套件與網路 API。

## 4. 介面資訊架構

頁面由上至下包含：

1. **標題區**
   - 標題：「胰島素與葡萄糖調節模擬器」
   - 說明：「加入葡萄糖或分泌胰島素，觀察血糖如何改變。」

2. **血糖監測卡片**
   - 主要數值，例如 `90 mg/dL`。
   - 狀態標籤，例如「正常範圍」。
   - 水平 Meter：範圍 `40–250 mg/dL`，以顏色區分低、正常、高。
   - Meter 上有會平滑移動的指標，並提供可讀的文字值。

3. **胰島素狀態**
   - 顯示相對量，例如 `胰島素：0 / 100`。
   - 可使用簡單進度條輔助呈現，但文字數值不可省略。

4. **操作區**
   - 按鈕 A：「A　攝取葡萄糖」；補充文字「血糖 +30 mg/dL」。
   - 按鈕 B：「B　分泌胰島素」；補充文字「胰島素 +25」。
   - 次要按鈕：「重新開始」。

5. **即時解說區**
   - 顯示最近發生的事件與因果關係。
   - 例：「攝取葡萄糖後，血糖上升了。」
   - 例：「胰島素正在作用，血糖逐漸下降。」

6. **頁尾提醒**
   - 「本模擬器使用簡化模型，僅供生物課程學習，不可用於醫療判斷。」

## 5. 視覺與 RWD 規格

### 5.1 視覺方向

- 風格清楚、友善、接近教學實驗工具。
- 背景與文字需有足夠對比。
- 不只依賴顏色傳達狀態；顏色旁必須同時顯示文字。
- 建議色彩：低血糖為藍色、正常為綠色、高血糖為橘紅色。
- 所有資訊應在單一頁面內完成，避免不必要的彈出視窗。

### 5.2 響應式行為

- 手機（小於 `600px`）：所有卡片與按鈕單欄排列，主要操作按鈕滿寬。
- 平板與電腦（大於等於 `600px`）：兩個主要操作按鈕可並排；內容區最大寬度約 `900px` 並置中。
- 頁面不可出現水平捲動。
- 觸控按鈕高度至少 `48px`，按鈕間保留足夠間距。
- 使用相對尺寸與 `clamp()` 調整標題及主要數字大小。

## 6. 模擬模型

此模型以「容易觀察因果」為優先，不追求臨床精確度。

### 6.1 狀態變數

```js
const INITIAL_STATE = {
  glucose: 90,   // 血糖，mg/dL
  insulin: 0,    // 教學用相對量，0–100
  running: true
};
```

數值限制：

- `glucose` 限制在 `40–250`。
- `insulin` 限制在 `0–100`。

### 6.2 操作規則

- 按下「攝取葡萄糖」：`glucose += 30`。
- 按下「分泌胰島素」：`insulin += 25`。
- 所有變更後均使用 `clamp` 限制範圍。
- 按鈕應可重複操作，讓學生自行探索極端狀態。

### 6.3 每秒更新規則

使用 `setInterval` 每 `1000ms` 更新一次：

```js
if (insulin > 0 && glucose > 40) {
  const glucoseDrop = Math.min(insulin * 0.08, glucose - 40);
  glucose -= glucoseDrop;
}

insulin = Math.max(0, insulin - 4);
```

解釋：胰島素越多，每秒降糖效果越明顯；胰島素也會隨時間衰減，因此作用是逐步發生且有限的。顯示血糖時四捨五入為整數，內部計算可保留小數。

MVP 不加入自動回復到 90 的機制，以免學生誤以為沒有胰島素時血糖也會快速自行下降。

### 6.4 血糖分類

此處為教學介面所用的簡化區間，不作診斷用途：

| 血糖值 | 顯示狀態 | 顏色建議 |
|---|---|---|
| `< 70` | 偏低 | 藍色 |
| `70–140` | 正常範圍 | 綠色 |
| `> 140` | 偏高 | 橘紅色 |

在 `glucose <= 50` 或 `glucose >= 220` 時，可將狀態文字加強為「非常低」或「非常高」，但不可加入醫療處置建議。

## 7. JavaScript 架構建議

保持單純、可讀，至少拆成以下函式：

```js
function clamp(value, min, max) {}
function addGlucose() {}
function releaseInsulin() {}
function updateSimulation() {}
function getGlucoseStatus(glucose) {}
function render() {}
function resetSimulation() {}
```

實作原則：

- 狀態資料是畫面的唯一來源，每次狀態改變後呼叫 `render()`。
- 不要把計算邏輯散落在多個事件監聽器中。
- DOM 元素在初始化時集中取得並保存參照。
- Meter 指標位置換算公式：

```js
const percent = ((glucose - 40) / (250 - 40)) * 100;
```

- 使用 CSS `transition` 讓指標平滑移動。
- 重新開始時重設狀態、即時解說與畫面；計時器只能維持一個，不可重複建立。

## 8. 無障礙與操作要求

- 使用語意化的 `<button>`，不可用 `<div>` 假裝按鈕。
- 按鈕需有清楚可見的鍵盤焦點樣式。
- 文字與背景對比至少符合 WCAG AA 的一般文字要求。
- Meter 使用適當的 ARIA meter 屬性：

```html
role="meter"
aria-valuemin="40"
aria-valuemax="250"
aria-valuenow="目前血糖值"
aria-valuetext="目前血糖值與狀態"
```

- 即時解說區可使用 `aria-live="polite"`，避免每秒更新的數值區頻繁打斷螢幕閱讀器。
- 動畫只使用必要的短暫轉場；透過 `prefers-reduced-motion` 關閉轉場。
- 可選擇支援鍵盤快捷鍵 `A`、`B`，但不可取代可點擊按鈕；使用者在表單控制項上輸入時不應誤觸快捷鍵。

## 9. 預期互動流程

### 情境一：觀察血糖上升

1. 初始血糖為 `90 mg/dL`，狀態為「正常範圍」。
2. 學生按下 A。
3. 血糖變為 `120 mg/dL`，Meter 指標向右移動。
4. 再按一次 A，血糖變為 `150 mg/dL`，狀態切換為「偏高」。

### 情境二：觀察胰島素作用

1. 血糖偏高時，學生按下 B。
2. 胰島素相對量增加 25。
3. 接下來每秒，血糖逐步下降，胰島素也逐步衰減。
4. 當血糖回到 `70–140 mg/dL`，狀態切換為「正常範圍」。

### 情境三：重新開始

1. 學生按下「重新開始」。
2. 血糖回到 `90 mg/dL`、胰島素回到 `0`。
3. Meter、狀態文字與解說同步恢復初始狀態。

## 11. 建議測試案例

1. 載入頁面，確認初始血糖 90、胰島素 0。
2. 連按 A 兩次，確認血糖為 150 且顯示偏高。
3. 在血糖 150 時按 B 一次，等待 3 秒，確認血糖逐步下降而非瞬間歸零。
4. 連按 A 多次，確認血糖不超過 250。
5. 連按 B 多次，確認胰島素不超過 100。
6. 在胰島素作用期間按「重新開始」，確認所有數值立即回到初始值，且之後只有單一計時器持續更新。
7. 將視窗縮至 360px，確認按鈕、Meter 與文字沒有被裁切。
8. 使用 Tab、Enter、Space 操作按鈕，確認功能正常。
9. 開啟 `prefers-reduced-motion`，確認 Meter 不使用平滑動畫。
